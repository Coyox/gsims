1        Introduction

This section will contain an introduction for the document and software.

1.1 Intended readership

This document is intended for users of the GSIMS software, including but not limited to students, teachers, and administrative staff of Gobind Sarvar school[a].  No technical knowledge is assumed or required for readers of this document.  For a more technical overview, refer to “Installation Guide & Technical Information.”

1.2 Purpose

The purpose of this user manual is to assist the user in learning to use the functionalities of the GSIMS software system.

1.3 How to use this document

This section will provide an overview of the sections and which user types will find each section most useful.

1.4 Related Documents

This section will point to any other documents that may be of use to the user’s learning of this system.

For a more technical description, see “Installation Guide & Technical Information.”

1.5 Constraints

1.5.1 Technology Constraints[b]

  • As the application is web-based, access is limited by the user’s ability to connect to the internet.  Possible issues include but are not limited to: poor WiFi connectivity, slow or unstable local network, Internet Service Provider outage, and web browser failure.
  • The service is hosted off-site by a third party and is subject to the stability and performance of their systems.
  • All email functionalities will rely on an external email service.  If, at any point, the service is down while there is an attempt to send an email, the application will alert the user.
  • The application is designed to run on the following web browsers:
  • Google Chrome version 14 and higher,
  • Microsoft Internet Explorer version 9 and higher,
  • Mozilla Firefox version 7 and higher, and
  • Apple Safari version 7 and higher.

Performance on browsers other than those listed above, including older versions of those listed above, is not tested and no guarantee is made as to the functionality of the application except on the supported browsers.

  • JavaScript must be enabled on the user’s web browser; the application cannot be accessed when it is disabled.
